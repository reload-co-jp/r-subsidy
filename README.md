# 中小企業向け補助金ポータルサイト R subsidy

## 1. 概要

### 1.1 サイトの目的
中小企業・個人事業主が、自社属性を先に入力することで、自社に合った補助金情報を絞り込んで閲覧できるポータルサイトを提供する。

本サイトは、単なる補助金一覧ではなく、以下を重視する。

- 自社向けの補助金を探しやすいこと
- 国の補助金に加えて東京都の補助金も探せること
- 難解な制度情報を平易に整理して表示すること
- 静的サイトとして低コスト・高速に運用できること

### 1.2 想定ユーザー
- 中小企業の経営者
- 小規模事業者
- 個人事業主
- 補助金を初めて探す担当者
- 東京都内の事業者、または東京都の制度にも関心がある事業者

### 1.3 提供価値
- 自社属性を入力すると、条件に合う補助金を優先表示できる
- 「全国対象」と「東京都対象」を同時に比較できる
- 補助金ごとの対象条件、用途、上限額、補助率、募集状況が一目でわかる
- JSONデータを更新するだけでサイト全体を更新できる

---

## 2. システム方針

### 2.1 基本方針
- DBは使わず、JSONファイルをデータソースとする
- サイトは静的生成する
- ユーザーの入力内容はブラウザ上で保持し、サーバー保存は行わない
- 検索・絞り込み・スコアリングはフロントエンド上で実行する

### 2.2 想定構成
- フロントエンド: Next.js または Astro
- データソース: JSONファイル
- 配信: GitHub Pages / Cloudflare Pages / Vercel など
- 更新方法: JSON編集 + ビルド + デプロイ

### 2.3 非対象
- 補助金の電子申請機能
- 会員登録機能
- 管理画面
- サーバーサイドDB
- 補助金の完全自動収集

## 3. サイト全体構成

### 3.1 ページ一覧
- トップページ
- 自社診断ページ
- 補助金一覧ページ
- 補助金詳細ページ
- 東京都補助金一覧ページ
- 使い方ページ
- 更新履歴ページ
- 免責事項ページ

### 3.2 導線
1. トップページにアクセス
2. 「自社に合う補助金を診断する」から自社属性を入力
3. マッチ度順に補助金を表示
4. 詳細ページで内容確認
5. 公式サイトへ遷移

## 4. 機能要件

## 4.1 自社属性入力機能

### 4.1.1 目的
ユーザーが自社情報を入力し、その情報に基づいて補助金を絞り込めるようにする。

### 4.1.2 入力項目
以下をMVPの入力項目とする。

- 事業形態
  - 法人
  - 個人事業主

- 所在地
  - 都道府県単位
  - 初期候補として「東京都」を選びやすくする

- 業種
  - IT・通信
  - 製造業
  - 建設業
  - 卸売業
  - 小売業
  - 飲食業
  - 宿泊業
  - サービス業
  - 医療・福祉
  - 教育
  - その他

- 従業員数
  - 1名
  - 2〜5名
  - 6〜20名
  - 21〜50名
  - 51名以上

- 創業年数
  - 1年未満
  - 1〜3年
  - 3〜5年
  - 5年以上

- 年商レンジ
  - 〜1,000万円
  - 1,000万〜5,000万円
  - 5,000万〜1億円
  - 1億円以上

- 利用目的（複数選択可）
  - IT導入
  - DX
  - 設備投資
  - 省力化
  - 販路開拓
  - 新商品開発
  - 新規事業
  - 人材育成
  - 採用
  - 省エネ
  - 創業

- 検討状況
  - 今すぐ申請したい
  - 情報収集中
  - 将来に向けて探したい

### 4.1.3 保存方式
- localStorage に保存
- ページ再訪時に復元
- ユーザーが「リセット」できるようにする

---

## 4.2 補助金一覧表示機能

### 4.2.1 目的
JSONデータから補助金を一覧表示する。

### 4.2.2 表示項目
一覧カードには以下を表示する。

- 補助金名
- 概要
- 対象地域
- 用途タグ
- 補助率
- 補助上限額
- 募集状況
- 締切日
- マッチ度
- 詳細ページへのリンク

### 4.2.3 並び順
初期表示は以下の優先順とする。

1. マッチ度が高いもの
2. 募集中のもの
3. 締切が近いもの
4. 更新日が新しいもの

### 4.2.4 フィルタ条件
一覧ページでは追加フィルタを利用可能とする。

- 全国 / 東京都
- 募集中のみ
- 用途
- 業種
- 事業形態
- 補助率
- 上限額帯
- 締切が近いもの

---
